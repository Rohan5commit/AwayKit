# Demo Script (2-4 minutes)

## Narration

### Opening (30s)
"AwayKit is a matchday coordination app for football fans. It combines local AI, peer-to-peer sync, and self-custodial wallets into one privacy-first workflow. Let me show you how it works."

### Demo Mode (30s)
"I'll open the demo with a pre-built scenario: 4 friends traveling to an away match. One click loads the full group, trip plan, expenses, and chat history."

### Trip Board (45s)
"Here's the shared trip board. You can see all group members with their arrival status, the shared fund balance of 500 USDT, the trip plan to the Santiago Bernabeu, and the checklist. Everything syncs peer-to-peer via Pears."

### AI Assistant (45s)
"Now let's use the local AI assistant powered by QVAC. I'll ask for a matchday checklist — notice this runs entirely on the device, no cloud API. I'll also translate a phrase into Spanish for the away match. All private, all local."

### Split & Settle (45s)
"For expenses, one person paid for the taxi. The app calculates each person's share automatically. I'll settle the pending amount using WDK — this sends real USDt from one self-custodial wallet to another. No trusted middleman needed."

### Architecture (15s)
"The architecture page explains how QVAC, Pears, and WDK work together. This is better than cloud chat plus centralized split-pay apps because it works offline, keeps data private, and removes the need for a trusted organizer."

### Closing (15s)
"AwayKit: move together, talk together, pay together. Private on-device AI, peer-to-peer sync, self-custodial USDt. Built for the Tether Developers Cup."

## Key Demo Points

- [x] One-click demo with no setup friction
- [x] QVAC generates checklist and translations locally
- [x] Pears handles group sync (simulated in demo)
- [x] WDK executes USDt settlements (simulated in demo)
- [x] Privacy story visible in every screen
- [x] Works offline (local-first architecture)
